# Critique Rubric (Anti-Anti-Ness)

A fill-in review sheet. Affirm first, cut precisely, then build. Drop preferences — they are noise unless asked for. Copy this template per review.

---

## KEEP — what works and must survive any revision
State what the work gets right before cutting anything. If you cannot name this, you do not yet understand the work well enough to critique it.

One block per finding. A finding earns a cut only if it causes a wrong result, a failure, or a misleading outcome. If it is "I'd have done it differently," it is a preference — drop it, do not list it.

## BUILD — the proposed next form
For each finding worth fixing, name the smallest change that resolves it without losing anything in KEEP. If two parts conflict, propose the synthesis that keeps both truths rather than picking a side.
