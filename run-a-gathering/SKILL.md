# Run a Gathering

Most runs are sub-agents (or steps) executing in sequence — output stapled to output, nothing integrated. A gathering is different: it *turns* — it becomes something that did not exist before it began — and it leaves a record for whoever runs next. Use this to structure your work toward an artifact, not just a transcript.

Read "gathering" as your run. You are the orchestrator. The participants are the sub-agents and roles you delegate to (or, if you are working solo, the steps and sub-tasks you move through). A human or mixed group is the same shape with people in those seats.

The CoCA Model — seven laws, in order:

## I. The Calling-Together
A gathering is recognized, not merely convened. Before you fan out work, let the real task surface instead of charging at your first framing of it. *The host speaks last; the gathering speaks first; the purpose arrives in the middle.* State the shared goal, then — for a multi-agent run — ask each sub-agent (or, solo, interrogate each major step) what it believes the real task is, and reconcile that against the goal before assigning anything.

## II. The Table of Witness
Designate the record before anything else — a shared scratchpad, a notes/memory file, a working doc the whole run reads and writes. Everything of consequence is written into it. The record is not bookkeeping; it is the first participant. In a multi-agent run this is the single source of truth sub-agents append to; solo, it is the file you externalize state into so you don't lose the thread across steps.

## III. The Ritual of Showing Up
Enumerate the run. Name every sub-agent and role and what each owns, once — out loud, in the record. Presence is the proof of continuity. The point is coverage: if a piece of the task has no named owner (no agent, no step), it will be silently dropped. Showing up is making sure nothing is unowned.

## IV. The Frictional Arc (the heart)
Move the run through three acts:
- **The Promise** — intentions stated plainly; what each sub-agent (or step) will attempt and return.
- **The Friction** — let results collide. Surface where sub-agents disagree, where one's output contradicts another's; do not average them or paper over the gap. Tension is evidence of vitality.
- **The Turning** — a synthesis that *neither* sub-result held at the start. Not picking a winner, not splicing both — a resolution produced by making them confront each other.

**Every gathering must turn.** If your run ends where it began — sub-agent outputs concatenated, no integration — it was not a gathering, just parallelism. As orchestrator, force the turn: put conflicting sub-results in the same context and require a reconciliation pass.

## V. The Artifact
Close by producing something durable: a decision, a spec, a diff, a written disagreement, a plan. *You do not choose the artifact; the artifact chooses you* — but you must capture it into the record. A run with no artifact did not happen; transient reasoning that wasn't written down is gone.

## VI. The Generational Clause
Write for the next run — a future agent or session with no memory of this one, a later you, a stranger in 2083. Leave coordinates: enough context in the record that a reader starting cold can understand what was decided and why, what was tried and rejected, what is still open. This is what separates a record from a residue.

## VII. The Exit
End deliberately. The run ends when the record is closed, not when the last sub-agent returns. Summarize the turn, file the artifact, and state the next obligation — and its owner (which agent, which run, or which human) — explicitly, so the next session inherits a hand-off, not a fog.

## Compact run checklist
```
[ ] Real task surfaced and reconciled against the goal (not just assumed)
[ ] Shared record/scratchpad designated and writable
[ ] Every sub-task owned by a named agent/step; nothing unowned
[ ] Promise → Friction → Turning (conflicting results actually reconciled)
[ ] Durable artifact captured into the record
[ ] Context left for the next run (a reader with no memory)
[ ] Closed with next obligation + owner
```
